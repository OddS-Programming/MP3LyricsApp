### Issue #1: Таймаут при медленном интернете

**Статус:** ЗАФИКСИРОВАНА
**Тяжесть:** Низкая
**Найдена:** 24 ноября 2025

**Описание:** При интернет соединении медленнее 1 Мбит/с поиск текста может превысить 15 секунд и таймаут

**Сценарий воспроизведения:**
1. Уменьшить скорость интернета (используя DevTools Chrome)
2. Попытаться поискать текст на Genius
3. Ожидание 20+ секунд → таймаут

**Ожидаемое поведение:** Ожидание текста или улучшенное сообщение об ошибке
**Текущее поведение:** Таймаут исключение
**Решение:** Увеличить таймаут с 15 до 30 секунд в v1.0.1
**Приоритет:** v1.0.1 (скорая)
**Автор:** User #3

### Issue #4: Поиск не работает на русском языке

**Статус:** НАЙДЕНА (НОВАЯ)
**Тяжесть:** Средняя
**Найдена:** 24 ноября 2025

**Описание:** При поиске музыки с русскими названиями артистов или треков приложение не находит текст на Genius, хотя текст там есть

**Примеры проблемных случаев:**
- Артист: "Артур Пирожков", Трек: "Кони"
- Артист: "Ленинград", Трек: "Экспонат"
- Артист: "Земфира", Трек: "Жить в эпоху перемен"

**Сценарий воспроизведения:**
1. Загрузить MP3 файл с русским названием
2. Выбрать опцию 4 (поиск текста)
3. Ввести русское имя артиста и трека
4. Результат: "Text not found" (хотя текст есть на сайте)

**Ожидаемое поведение:** Текст должен быть найден на Genius даже для русских исполнителей
**Текущее поведение:** Функция поиска не работает с кириллицей (неправильное кодирование URL)
**Причина:** Проблема с кодированием русских символов при формировании URL для Genius
**Решение:** Использовать UTF-8 кодирование при формировании URL, добавить URL-encoding для специальных символов
**Рекомендуемый код:**
```java
String encodedArtist = URLEncoder.encode(artist, "UTF-8");
String encodedTrack = URLEncoder.encode(track, "UTF-8");
String url = String.format("https://genius.com/api/search/multi?per_page=5&q=%s%%20%s", encodedArtist, encodedTrack);
```

**Приоритет:** HIGH (важно для пользователей из России)
**Версия для исправления:** v1.0.1 (срочно)
**Автор:** QA Team
